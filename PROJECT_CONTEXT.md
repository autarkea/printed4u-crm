## 🎯 ЦЕЛЬ ПРОЕКТА

Создать **коробочную CRM-систему** для малого бизнеса (3D-печать, инжиниринг), которая:
- Работает на **собственном мини-компьютере** (Ubuntu 24 Server) или VPS
- Использует **NocoDB CE** как базу данных (встроенный в docker-compose)
- Интегрируется с **Telegram** для управления задачами
- Генерирует **PDF-документы** (счета, акты, накладные)
- Автоматизирует **создание папок проектов**
- Отправляет **email с PDF** клиентам
- Делает **бэкапы** (локальные + облачные)

**Ключевое требование:** Система должна быть **портативной** — её можно развернуть на любом сервере без привязки к конкретному домену/IP.

# NocoDB
NOCO_URL=http://nocodb:8080  # ✅ Внутренний Docker-адрес
NOCO_TOKEN=your_noco_token_here
BASE_ID=your_base_id_here

# ID таблиц (заполняются автоматически при установке!)
TABLE_TASKS=
TABLE_CONTACTS=
TABLE_PROJECTS=
TABLE_DOCUMENTS=
TABLE_ITEMS=
TABLE_LEGAL_ENTITIES=
TABLE_MY_DETAILS=

# Email (SMTP)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_app_password
SMTP_FROM=your_email@gmail.com

# Webhook
WEBHOOK_SECRET=your_secret_here

# NocoDB JWT (генерируется автоматически при установке)
JWT_SECRET=auto_generated_base64_string

# System
TZ=Europe/Minsk

**ВАЖНО:**
- `NOCO_URL` должен быть `http://nocodb:8080` (внутренний Docker-адрес)
- `BASE_ID` — только ID базы (например, `ptkx1tv0v3334vf`), без workspace ID
- ID таблиц заполняются автоматически через `template.db`

## 🧠 КЛЮЧЕВЫЕ ИНСАЙТЫ NocoDB CE

### 1. Таблица workspace (НЕ nc_org!)
NocoDB CE использует **СТАРУЮ** таблицу `workspace`, а не новую `nc_org`:
```sql
-- ПРАВИЛЬНО:
SELECT id FROM workspace;
-- НЕПРАВИЛЬНО:
SELECT id FROM nc_org;  -- всегда пусто в CE!
```

### 2. Workspace создаётся только при создании базы
NocoDB **НЕ создаёт** workspace автоматически при регистрации! Workspace создаётся только когда пользователь создаёт **первую базу** (не демо "Getting Started").

### 3. NC_DEFAULT_WORKSPACE_ID
Если в базе есть `NC_DEFAULT_WORKSPACE_ID` со старым ID, NocoDB думает, что workspace уже есть, и не создаёт новый. Нужно **удалять** эту запись из template.db.

### 4. fk_workspace_id в 25+ таблицах
Workspace ID хранится не только в `nc_bases_v2`, но и в:
- `nc_models_v2`, `nc_views_v2`, `nc_columns_v2`
- `nc_grid_view_columns_v2`, `nc_gallery_view_columns_v2`
- `nc_kanban_view_columns_v2`, `nc_form_view_columns_v2`
- `nc_col_relations_v2`, `nc_col_select_options_v2`
- `nc_filter_exp_v2`, `nc_sort_v2`, `nc_hooks_v2`
- ... и ещё 13 таблицах

**Все эти таблицы нужно обновлять** при применении шаблона!

### 5. Пустые модели
В template.db могут быть модели без `table_name` (например, "Untitled", "Структура таблиц"). Их нужно **удалять**, иначе NocoDB показывает пустые таблицы.

## 🐛 ИЗВЕСТНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Проблема 1: Hardcoded секреты в коде (решено 28 июня)
**Было:** В коде были захардкожены токены, ID таблиц, домены, IP  
**Решение:** Все секреты вынесены в `.env`, в коде используются `process.env.*`

### Проблема 2: Кавычки вокруг process.env (решено 28 июня)
**Было:** `NOCO_TOKEN: 'process.env.NOCO_TOKEN'` (строка!)  
**Решение:** Убраны кавычки: `NOCO_TOKEN: process.env.NOCO_TOKEN`

### Проблема 3: Неправильная подстановка в URL (решено 28 июня)
**Было:** `${NOCO_API_URL}/${BASE_ID}/process.env.TABLE_MY_DETAILS/1`  
**Решение:** `${NOCO_API_URL}/${BASE_ID}/${process.env.TABLE_MY_DETAILS}/1`

### Проблема 4: Сломанный синтаксис стрелочных функций (решено 28 июня)
**Было:** `= >` (с пробелом) вместо `=>`  
**Решение:** Исправлено во всех файлах

### Проблема 5: Dockerfile копирует несуществующие файлы (решено 30 июня)
**Было:** `COPY server.js bot.js config.js schema.js start.sh .env ./`  
**Решение:** Убраны несуществующие файлы, `.env` не копируется в образ

### Проблема 6: Docker не может копировать из parent directory (решено 30 июня)
**Было:** `COPY ../templates /app/templates`  
**Решение:** Изменён build context на корень проекта

### Проблема 7: NocoDB не запускается — "Database is not supported" (решено 30 июня)
**Было:** `NC_DB=sqlite:///usr/app/data/nocodb.db`  
**Решение:** Убрана переменная `NC_DB`, NocoDB использует SQLite по умолчанию

### Проблема 8: Healthcheck использует wget (решено 30 июня)
**Было:** `test: ["CMD", "wget", ...]`  
**Решение:** Заменено на `test: ["CMD", "curl", "-f", "http://localhost:8080"]`

### Проблема 9: setup.js использует неправильный API endpoint (решено 30 июня)
**Было:** `/api/v1/db/meta/bases/{baseId}/tables`  
**Решение:** NocoDB v2 API — `/api/v2/meta/bases/{baseId}/tables`

### Проблема 10: Права доступа к /opt/printed4u-crm (решено 30 июня)
**Было:** `error: cannot open '.git/FETCH_HEAD': Permission denied`  
**Решение:** Добавлено `sudo chown -R $USER:$USER $INSTALL_DIR`

### Проблема 11: Workspace не создаётся (решено 2 июля) 🔥
**Было:** NocoDB не создаёт workspace автоматически  
**Решение:** Пользователь должен создать базу "temp" вручную → workspace создаётся

### Проблема 12: apply-template.sh не находит workspace (решено 2 июля) 🔥
**Было:** Скрипт искал workspace в `nc_org` (пустая таблица!)  
**Решение:** Искать в таблице `workspace` (старая схема NocoDB CE)

### Проблема 13: Таблицы не видны в UI (решено 2 июля) 🔥
**Было:** `fk_workspace_id` в моделях указывал на старый workspace  
**Решение:** Обновлять `fk_workspace_id` во **ВСЕХ** таблицах (25+ штук!)

### Проблема 14: Колонки не отображаются (решено 2 июля) 🔥
**Было:** `fk_workspace_id` в `nc_columns_v2` указывал на старый workspace  
**Решение:** Обновлять `fk_workspace_id` в `nc_columns_v2`

### Проблема 15: База CRM пустая в UI (решено 2 июля) 🔥
**Было:** В template.db были метаданные старых баз (Заказы, Getting Started)  
**Решение:** Удалять все модели, источники, базы кроме CRM

### Проблема 16: NocoDB токен пропадает (решено 2 июля) 🔥
**Было:** `apply-template.sh` очищал `nc_api_tokens`  
**Решение:** Пользователь создаёт токен ПОСЛЕ применения шаблона

### Проблема 17: BASE_ID не устанавливается (решено 2 июля) 🔥
**Было:** В `.env` оставался плейсхолдер `your_base_id_here`  
**Решение:** `install.sh` автоматически определяет BASE_ID после применения шаблона

- ✅ Полный контроль
- ✅ Работает без интернета (локально)

**Минусы:**
- ❌ Нужен белый IP (5-15$/мес у провайдера)
- ❌ Нужен домен (10-20$/год)
- ❌ Нужен проброс портов 80 и 443 на роутере

### Вариант 2: VPS (для теста) 🧪

**Для кого:** Тестирование, очень маленькие компании

**Провайдеры:**
- **Hetzner** (Германия) — от 4€/мес (2GB RAM)
- **Timeweb** (Россия) — от 300₽/мес
- **Selectel** (Россия) — от 400₽/мес

**Плюсы:**
- ✅ Быстрый старт (5 минут)
- ✅ Белый IP включён
- ✅ 24/7 без обслуживания

**Минусы:**
- ❌ Абонентская плата
- ❌ Данные на чужом сервере
- ❌ Для юрлиц — вопрос хранения данных

### Вариант 3: Арендованный сервер (для крупных клиентов) 🏢

**Для кого:** Активные компании с большим объёмом данных

**Провайдеры:**
- **Hetzner Dedicated** — от 35€/мес (Intel i3, 32GB RAM)
- **Timeweb Dedicated** — от 2500₽/мес

**Плюсы:**
- ✅ Мощный и надёжный
- ✅ SLA от провайдера
- ✅ Можно хранить большие данные

**Минусы:**
- ❌ Дороже (35-100€/мес)
- ❌ Избыточно для малых компаний

### Cloudflare Tunnel (опционально) 🔥

**Для кого:** Клиенты без белого IP, которые хотят HTTPS

**Как работает:**
```
Интернет → Cloudflare → Tunnel → Мини-ПК (за NAT)
```

**Плюсы:**
- ✅ Не нужен белый IP
- ✅ Не нужна настройка роутера
- ✅ Автоматический HTTPS
- ✅ Бесплатно

**Минусы:**
- ❌ Зависимость от Cloudflare
- ❌ Клиент сам регистрирует аккаунт

**Решение:** Cloudflare Tunnel — опциональная настройка, клиент делает сам или за доп. плату.

## 🔒 HTTPS И ДОМЕНЫ

### Архитектура (Nginx + Let's Encrypt)
```
Интернет → Nginx (443) → NocoDB (localhost:8081)
                        → PDF Generator (localhost:3000)
                        → Webhook (localhost:3001)
```

### Требования
1. ✅ **Домен** (~10-20$/год) — например, `crm.client.by`
2. ✅ **Белый (статический) IP** — от провайдера (5-15$/мес)
3. ✅ **Проброс портов** 80 и 443 на роутере
4. ✅ **DNS** — A-запись домена на IP сервера

### Автоматизация
При установке через `install.sh`:
- ✅ Спрашивает домен
- ✅ Устанавливает Nginx + Certbot
- ✅ Получает SSL сертификат через Let's Encrypt
- ✅ Настраивает прокси на все сервисы
- ✅ Сертификат обновляется автоматически

### Почему HTTPS обязателен
- ❌ Без HTTPS **Telegram webhook не работает** (Telegram требует HTTPS)
- ❌ Пароли и токены передаются открытым текстом
- ❌ Современные функции браузера недоступны
- ❌ Браузер показывает "Небезопасное соединение"
- ❌ Юридические риски (закон о персональных данных)

## 💡 СОВЕТЫ ДЛЯ БУДУЩИХ СЕССИЙ

1. **Всегда проверяй `.env.example`** — если добавляешь новую переменную, добавь её в шаблон
2. **Никаких hardcoded секретов** — всё через `process.env.*`
3. **Тестируй на чистом сервере** — код должен работать без привязки к конкретному домену/IP
4. **Документируй изменения** — обновляй этот файл при значительных изменениях
5. **Используй Docker** — все компоненты должны запускаться через `docker-compose`
6. **NocoDB API** — используй v2 API (`/api/v2/...`), v1 устарел
7. **Внутренние Docker-адреса** — используй `http://nocodb:8080`, а не `http://localhost:8081`
8. **Права доступа** — всегда проверяй права на `/mnt/data` и `/opt/printed4u-crm`
9. **NocoDB CE архитектура** — используй таблицу `workspace`, а не `nc_org`
10. **fk_workspace_id** — обновляй во ВСЕХ таблицах при применении шаблона
11. **NC_DEFAULT_WORKSPACE_ID** — удаляй из template.db, иначе workspace не создастся
12. **Пустые модели** — удаляй модели без `table_name` из template.db
13. **HTTPS обязателен** — без него Telegram webhook не работает
