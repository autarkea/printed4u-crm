## 🎯 ЦЕЛЬ ПРОЕКТА

Создать **коробочную CRM-систему** для малого бизнеса (3D-печать, инжиниринг), которая:
- Работает на **собственном мини-компьютере** (Ubuntu 24 Server)
- Использует **NocoDB** как базу данных
- Интегрируется с **Telegram** для управления задачами
- Генерирует **PDF-документы** (счета, акты, накладные)
- Автоматизирует **создание папок проектов**
- Отправляет **email с PDF** клиентам
- Делает **бэкапы** (локальные + облачные)

**Ключевое требование:** Система должна быть **портативной** — её можно развернуть на любом сервере без привязки к конкретному домену/IP.

### Компоненты:

1. **Telegram Bot** (`bot/`)
   - Управление задачами, контактами, проектами
   - Приём пересланных сообщений от клиентов
   - Утренние рассылки и напоминания о дедлайнах

2. **PDF Generator** (`bot/server.js`)
   - Генерация PDF из HTML-шаблонов
   - Отправка email с PDF
   - Прокси-роуты для HTML-шаблонов (безопасный доступ к NocoDB API)

3. **Project Webhook** (`webhook/server.js`)
   - Автоматическое создание папок проектов
   - Создание symlink'ов для документов
   - Генерация Client ID для клиентов

4. **NocoDB** (внешняя система)
   - База данных (таблицы: Tasks, Contacts, Projects, Documents, Items, Legal Entities, My Details)
   - Веб-интерфейс для управления данными

## 🔐 ПЕРЕМЕННЫЕ ОКРУЖЕНИЯ (.env)

Все секреты и конфигурация вынесены в `.env` файл. Шаблон в `.env.example`:

```bash
# Telegram
TELEGRAM_BOT_TOKEN=your_bot_token_here
TELEGRAM_USER_ID=your_telegram_id_here

# NocoDB
NOCO_URL=http://host.docker.internal:8081/api/v1/db/data
NOCO_TOKEN=your_noco_token_here
BASE_ID=your_base_id_here

# ID таблиц (заполняются автоматически при установке!)
TABLE_TASKS=
TABLE_CONTACTS=
TABLE_PROJECTS=
TABLE_DOCUMENTS=
TABLE_ITEMS=
TABLE_LEGAL_ENTITIES=
TABLE_MY_DETAILS=

# Email (SMTP)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_app_password
SMTP_FROM=your_email@gmail.com

# Webhook
WEBHOOK_SECRET=your_secret_here

# System
TZ=Europe/Minsk
```

**ВАЖНО:** ID таблиц (`TABLE_*`) должны заполняться **автоматически** при установке через `install.sh` (пока не реализовано).

## 🎯 КЛЮЧЕВЫЕ ФУНКЦИИ

### 1. Управление задачами
- `/new` — создать задачу
- `/tasks` — список задач
- `/today` — задачи на сегодня
- `/history` — история выполненных задач
- Напоминания за 24 часа и 2 часа до дедлайна

### 2. Управление контактами
- `/add_contact` — добавить контакт (мастер из 5 шагов)
- `/contacts` — список контактов
- Обработка пересланных сообщений от клиентов
- Поиск дубликатов по телефону/username

### 3. Управление проектами
- `/project` — создать проект (мастер из 3 шагов)
- Автоматическое создание папок проектов с подпапками (Исходники, Макеты, Документы)
- Создание symlink'ов для PDF документов
- Генерация Client ID для клиентов

### 4. Генерация документов
- Генерация PDF из HTML шаблонов
- Поддержка подписи/печати (суффикс `_notsigned`)
- Отправка email с PDF вложением
- Выбор получателя (Контакт или Юрлицо)

### 5. Бэкапы
- Локальные бэкапы NocoDB (`/mnt/data/backups/`)
- Облачные бэкапы через rclone (Google Drive)
- Команда `/backup` — статус бэкапов

### 6. Утренние рассылки
- Ежедневная сводка в 10:00 (настраивается через `MORNING_CRON`)
- Список активных задач, проектов, контактов
- Статус бэкапов

## 🐛 ИЗВЕСТНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Проблема 1: Hardcoded секреты в коде
**Было:** В коде были захардкожены:
- Токены NocoDB (`nc_pat_...`)
- ID таблиц (`maob70r6njy4b31`, `mel0fql6jhtknu6`, `mh4tuppyvnapu7b`)
- Домены (`noco.printed4u.by`, `webhook.printed4u.by`)
- IP адреса (`192.168.2.200`)
- Имя пользователя (`Саня`)
- Пути (`/home/allexmea/`)

**Решение:** Все секреты вынесены в `.env`, в коде используются `process.env.*`

### Проблема 2: Кавычки вокруг process.env
**Было:** В `bot/config.js` были кавычки:
```javascript
NOCO_TOKEN: 'process.env.NOCO_TOKEN',  // ❌ СТРОКА!
```

**Решение:** Убраны кавычки:
```javascript
NOCO_TOKEN: process.env.NOCO_TOKEN,  // ✅ ПРАВИЛЬНО!
```

### Проблема 3: Неправильная подстановка в URL
**Было:**
```javascript
`${NOCO_API_URL}/${BASE_ID}/process.env.TABLE_MY_DETAILS/1`  // ❌
```

**Решение:**
```javascript
`${NOCO_API_URL}/${BASE_ID}/${process.env.TABLE_MY_DETAILS}/1`  // ✅
```

### Проблема 4: Сломанный синтаксис стрелочных функций
**Было:** `= >` (с пробелом) вместо `=>`

**Решение:** Исправлено во всех файлах

### Проблема 5: Hardcoded домены в email
**Было:**
```javascript
from: `"printed4u.by" <${process.env.SMTP_FROM}>`
```

**Решение:**
```javascript
from: `"CRM" <${process.env.SMTP_FROM}>`
```
