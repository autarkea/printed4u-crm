## 🎯 ЦЕЛЬ ПРОЕКТА

Создать **коробочную CRM-систему** для малого бизнеса (3D-печать, инжиниринг), которая:
- Работает на **собственном мини-компьютере** (Ubuntu 24 Server)
- Использует **NocoDB** как базу данных
- Интегрируется с **Telegram** для управления задачами
- Генерирует **PDF-документы** (счета, акты, накладные)
- Автоматизирует **создание папок проектов**
- Отправляет **email с PDF** клиентам
- Делает **бэкапы** (локальные + облачные)

**Ключевое требование:** Система должна быть **портативной** — её можно развернуть на любом сервере без привязки к конкретному домену/IP.

## 🐛 ИЗВЕСТНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Проблема 1: Hardcoded секреты в коде
**Было:** В коде были захардкожены:
- Токены NocoDB (`nc_pat_...`)
- ID таблиц (`maob70r6njy4b31`, `mel0fql6jhtknu6`, `mh4tuppyvnapu7b`)
- Домены (`noco.printed4u.by`, `webhook.printed4u.by`)
- IP адреса (`192.168.2.200`)
- Имя пользователя (`Саня`)
- Пути (`/home/allexmea/`)

**Решение:** Все секреты вынесены в `.env`, в коде используются `process.env.*`

### Проблема 2: Кавычки вокруг process.env
**Было:** В `bot/config.js` были кавычки:
```javascript
NOCO_TOKEN: 'process.env.NOCO_TOKEN',  // ❌ СТРОКА!
```

**Решение:** Убраны кавычки:
```javascript
NOCO_TOKEN: process.env.NOCO_TOKEN,  // ✅ ПРАВИЛЬНО!
```

### Проблема 3: Неправильная подстановка в URL
**Было:**
```javascript
`${NOCO_API_URL}/${BASE_ID}/process.env.TABLE_MY_DETAILS/1`  // ❌
```

**Решение:**
```javascript
`${NOCO_API_URL}/${BASE_ID}/${process.env.TABLE_MY_DETAILS}/1`  // ✅
```

### Проблема 4: Сломанный синтаксис стрелочных функций
**Было:** `= >` (с пробелом) вместо `=>`

**Решение:** Исправлено во всех файлах

### Проблема 5: Hardcoded домены в email
**Было:**
```javascript
from: `"printed4u.by" <${process.env.SMTP_FROM}>`
```

**Решение:**
```javascript
from: `"CRM" <${process.env.SMTP_FROM}>`
```

## 🐛 ИЗВЕСТНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Проблема 1: Hardcoded секреты в коде (решено 28 июня)
**Было:** В коде были захардкожены токены, ID таблиц, домены, IP  
**Решение:** Все секреты вынесены в .env, в коде используются `process.env.*`

### Проблема 2: Кавычки вокруг process.env (решено 28 июня)
**Было:** `NOCO_TOKEN: 'process.env.NOCO_TOKEN'` (строка!)  
**Решение:** Убраны кавычки: `NOCO_TOKEN: process.env.NOCO_TOKEN`

### Проблема 3: Неправильная подстановка в URL (решено 28 июня)
**Было:** `${NOCO_API_URL}/${BASE_ID}/process.env.TABLE_MY_DETAILS/1`  
**Решение:** `${NOCO_API_URL}/${BASE_ID}/${process.env.TABLE_MY_DETAILS}/1`

### Проблема 4: Сломанный синтаксис стрелочных функций (решено 28 июня)
**Было:** `= >` (с пробелом) вместо `=>`  
**Решение:** Исправлено во всех файлах

### Проблема 5: Dockerfile копирует несуществующие файлы (решено 30 июня)
**Было:** `COPY server.js bot.js config.js schema.js start.sh .env ./`  
**Решение:** Убраны несуществующие файлы, `.env` не копируется в образ (используется `env_file`)

### Проблема 6: Docker не может копировать из parent directory (решено 30 июня)
**Было:** `COPY ../templates /app/templates`  
**Решение:** Изменён build context на корень проекта, пути обновлены

### Проблема 7: NocoDB не запускается — "Database is not supported" (решено 30 июня)
**Было:** `NC_DB=sqlite:///usr/app/data/nocodb.db`  
**Решение:** Убрана переменная `NC_DB`, NocoDB использует SQLite по умолчанию

### Проблема 8: Healthcheck использует wget, которого нет в образе (решено 30 июня)
**Было:** `test: ["CMD", "wget", ...]`  
**Решение:** Заменено на `test: ["CMD", "curl", "-f", "http://localhost:8080"]`

### Проблема 9: setup.js использует неправильный API endpoint (решено 30 июня)
**Было:** `/api/v1/db/meta/bases/{baseId}/tables`  
**Решение:** NocoDB v2 API — `/api/v2/meta/bases/{baseId}/tables`

### Проблема 10: Права доступа к папке /opt/printed4u-crm (решено 30 июня)
**Было:** `error: cannot open '.git/FETCH_HEAD': Permission denied`  
**Решение:** Добавлено `sudo chown -R $USER:$USER $INSTALL_DIR` в install.sh

## 💡 СОВЕТЫ ДЛЯ БУДУЩИХ СЕССИЙ

1. **Всегда проверяй `.env.example`** — если добавляешь новую переменную окружения, добавь её в шаблон

5. **Используй Docker** — все компоненты должны запускаться через `docker-compose`
6. **NocoDB API** — используй v2 API (`/api/v2/...`), v1 устарел
7. **Внутренние Docker-адреса** — используй `http://nocodb:8080`, а не `http://localhost:8081`
8. **Права доступа** — всегда проверяй права на `/mnt/data` и `/opt/printed4u-crm`
